# AI技術構造リファレンス：探索から生成まで

## 1. AI能力の進化系統樹
AIは「解き方を教わる」フェーズから、「自らルールを抽出する」フェーズへ進化しました。

* **探索・論理（第1〜2次ブーム）**: 迷路の最短ルート計算や `if-then` ルール（エキスパートシステム）。
* **識別（2010年代〜）**: 特徴量を抽出し、境界線を引く。データに「ラベル（正解）」を貼って学習する（教師あり学習）。
* **生成（現在）**: データの「分布」そのものを学習し、未知のデータを生み出す（LLM、拡散モデル）。

---

## 2. 学習パイプラインの3大要素
現代のAI知能を形作る、データの処理プロセスです。

1.  **マスキング / ノイジング（破壊と自習）**: データの一部を欠損させ、それを復元させることで、人間が介在せずに「構造」を学ばせる手法。
2.  **クラスタリング（概念形成）**: 学習過程で、内部のベクトル空間において意味の近いもの（例：「リンゴ」と「バナナ」）が自然に近くに集まる現象。
3.  **ラベリング（意味付けと調整）**: 自習を終えたAIに対し、人間が「これは正しい回答だ」とラベルを与えてマナーや専門性を教える工程（ファインチューニング）。

---

## 3. アーキテクチャの比較：LLM vs 拡散モデル
扱うデータの性質によって、最適な「破壊と再生」の手法が使い分けられています。

| 項目 | **LLM (Large Language Models)** | **拡散モデル (Diffusion Models)** |
| :--- | :--- | :--- |
| **データの性質** | **離散（Discrete）**: 記号、文字、コード。 | **連続（Continuous）**: 画像、音声、動画。 |
| **破壊手法** | **マスキング**: 一部を「虫食い」にする。 | **ノイジング**: 全体に「砂嵐」をまぶす。 |
| **復元プロセス** | 次の記号を**一発予測**。 | 段階的に**デノイズ（掃除）**して磨き上げる。 |
| **設計思想** | 1文字の間違いが致命的な「パズル」。 | 全体の整合性が重要な「彫刻」。 |



---

## 4. 【補足】「識別（ラベル）」と「生成（拡散）」はどう繋がるのか？

「画像に何が写っているか」という識別と、「画像を生成する」拡散モデルのつながりは、**「条件付け（Conditioning）」**という概念で統合されます。

### なぜ「猫」の絵が出てくるのか？
1.  **識別の力（ラベリング）**: 
    AIは学習時、膨大な「画像」と「説明文（ラベル）」のセットを読み込みます。ここで、内部のベクトル空間において「猫の画像」と「猫という単語」が同じ位置に配置（クラスタリング）されます。
2.  **生成の力（拡散・デノイズ）**: 
    あなたが「猫の絵を描いて」と命じると、AIは「猫」という言葉に対応するベクトル位置を参照します。
3.  **融合**: 
    AIは**「この砂嵐（ノイズ）の中から、『猫』というラベルの概念に一致する形だけを残して掃除せよ」**という制約条件のもとでデノイズを行います。

> **エンジニア的解釈**: 拡散モデルにおける「猫」という指示は、デノイズ関数に対する**「バイアス（重み付け）」**として機能します。何もないノイズから特定のパターンを浮かび上がらせるための「ガイド役」がラベルなのです。

## 5. 応用技術のメカニズム
* **消しゴムマジック**: マスキング（修正範囲の特定）と、周辺の文脈から「あるべき背景」を予測する生成AI（インペインティング）のハイブリッド。
* **マルチモーダル**: テキスト、画像、音声を、内部では共通の「多次元ベクトル」として処理。これにより「画像という非構造データ」に「言葉という構造」を紐付けることが可能になった。
